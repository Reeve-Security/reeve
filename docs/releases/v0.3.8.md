## What's new since v0.3.7

v0.3.8 is the public-cut release. It promotes the validated
`v0.3.8-rc.4` launch candidate without changing the AIBOM schema
version. It focuses on launch-surface correctness, report readability,
redaction hardening, and reproducible output contracts.

- **Human-readable per-machine report.** The HTML report now starts with
  an executive summary, shows policy findings before raw tables, merges
  sensitive-data findings from the sibling report, rolls machine join keys
  into human counts, and keeps `#instance-*` identifiers out of the default
  view.
- **Redaction hardening across launch artifacts.** Mac, Windows, and Linux
  release-candidate runs proved zero raw endpoint usernames in the shipped
  AIBOM, sensitive-data report, SARIF, and HTML report paths used for the
  launch demo packet.
- **CycloneDX 1.5 conformance guard.** Emitted CDX files validate against
  the official CycloneDX 1.5 JSON Schema in CI, including scoped npm purls,
  dependencies, hashes, and external references.
- **Precise MCP registry matching.** Registry lookup output distinguishes
  exact purl matches, candidates, ambiguous results, no-match facts, and
  not-applicable synthetic components without upgrading fuzzy hits into
  match claims.
- **Release-boundary cleanup.** Reeve Labs pipeline producers, generated
  data, private publication workflows, and cross-repo tokens remain outside
  the public Reeve binary repository. Public registry fetch/seed/verify
  primitives stay because they are part of reproducibility.

## Install

After the repository is public, use the signed shell installer:

```bash
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/Reeve-Security/reeve/releases/latest/download/aibom-cli-installer.sh | sh
```

Or download an archive and verify it before use:

```bash
TAG=v0.3.8
ASSET=aibom-cli-x86_64-unknown-linux-gnu.tar.xz
BASE="https://github.com/Reeve-Security/reeve/releases/download/${TAG}"

curl -LO "${BASE}/${ASSET}"
curl -LO "${BASE}/${ASSET}.bundle"

cosign verify-blob \
  --bundle "${ASSET}.bundle" \
  --certificate-identity-regexp '^https://github.com/Reeve-Security/reeve/.github/workflows/release.yml@refs/tags/v[0-9]+\.[0-9]+\.[0-9]+.*$' \
  --certificate-oidc-issuer "https://token.actions.githubusercontent.com" \
  "${ASSET}"
```

Windows uses the same verifier, replacing `ASSET` with
`aibom-cli-x86_64-pc-windows-msvc.zip`.

## Launch validation

`v0.3.8-rc.4` was validated as the launch candidate before promotion:

- Windows signed zip, clean download, SYSTEM scan: zero raw username,
  zero PEM leak, report sections present, no `#instance-*` in the default
  HTML view.
- macOS released tarball scan: report rendered, zero raw username,
  sensitive-data summary merged.
- Linux released tarball scan: grants and redacted secret findings emitted.
- Grype consumed Reeve's CycloneDX output as-is and reported the expected
  vulnerability findings from the generated CDX.
- Labs registry lookup demo returned exact purl matches only where package
  coordinates matched; candidate and ambiguous states stayed explicitly
  non-claiming.

## Known limitations

- Windows profiling remains observational only. AppContainer or other
  Windows sandbox enforcement is not shipped.
- Cowork approvals remain bounded by the safe plaintext evidence Reeve can
  parse. Reeve does not decrypt approval blobs, extract tokens, or infer
  hidden connector grants.
- Conversation-log scanning remains opt-in. Metadata inventory and secret
  pattern scanning are separate flags and never serialize raw conversation
  content or raw secret values.
- Validator issue #479 is a post-cut hardening item: the signed-bundle path
  requires SHA-256 DSSE subjects, but the CDX-side externalReference hash
  check will be tightened to reject missing SHA-256 entries.
